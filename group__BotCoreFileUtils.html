bot_fileutils_get_unique_filename:

Returns an unused filename that can be used for writing without fear of overwriting another file. The returned string must be freed with free() when no longer needed. The new file is not opened. The filename is constructed as path/[date-]basename.XX.extension where XX is an integer chosen to make it unique. date- is only included if is 1.

: The path to the directory where the file should be stored. Pass NULL to use the current working directory or if the path is already included in . : The desired name of the file without any trailing extension. This may include a leading path, but only if is zero. : Set to 1 if the basename should be prefixed with the current date in the format YYYY-MM-DD-. : Filename extension to be used. Use NULL if no extension is desired.
